# Text Elements
1. j第一次遇到 -1 时，无法立马找出 (-1, -1, 2) 这个元组，因为右边只有一个 right 指针，
无法同时指到 -1 和 2. ^kaQsFaCC

2. j 第二次遇到 -1 时，如果直接跳过重复元素，则会遗失 (-1, -1, 2) 这个解；
但如果不跳过，则其它重复解又无法去除。 ^1PG18rts

由于每个指针都必须具备消重的能力，因此不能使用“固定 j 然后在其两侧寻找 l 和 r”的策略。
以下例子可以帮助理解这一点。 ^Y9Dp7Z5c

对于数组 [0, 0, 0, 0]，虽然 i、k 可以消重，可由于 j 不具备消重能力，
所以最终找出的答案为 `[[0, 0, 0], [0, 0, o]]`。 ^vDA8gvVl

比如把原来的 0 改为 -1，则不跳过则无法消去第二个 (-1, -1, 2)。 ^s3Sg3o7C
